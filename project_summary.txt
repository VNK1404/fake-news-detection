### Project Architecture (high‑level)
```mermaid
flowchart TD
    A[User submits image / PDF] --> B[OCR → extract raw text]
    B --> C[clean_text]
    C --> D[validate_text]
    D --> E[extract_main_claim]
    E --> F[Parallel API calls]
    subgraph APIs
        G[Gemini]
        H[Google Fact‑Check]
        I[NewsAPI]
        J[Guardian]
    end
    F --> G & H & I & J
    G --> K[raw label]
    H --> K
    I --> K
    J --> K
    K --> L[normalize_result (→ score -1..1)]
    L --> M[Weighted sum (API_WEIGHTS)]
    M --> N[final_decision (Real/Fake/Uncertain + confidence)]
    N --> O[JSON response + UI rendering]
```
*The pipeline (`pipeline.py`) orchestrates steps **A–O**. The decision engine (`decision_engine.py`) handles the scoring (`M`) and final decision (`N`).*

### 1. **Set environment variables**:
   ```powershell
   $Env:GEMINI_API_KEY="your-gemini-api-key"
   $Env:GUARDIAN_API_KEY="your-guardian-api-key"
   ```
2. **Run the diagnostic script** to sanity‑check all APIs:
   ```powershell
   python check_api_status.py
   ```
   - Output shows each API status (see the latest run above). Gemini currently returns *Unknown* due to a 503; retry later.
3. **Start the Flask server** (if you want the web UI):
   ```powershell
   python app.py
   ```
   - Visit `http://127.0.0.1:5000` in a browser, upload a file, and view the claim, API labels, and final decision.
4. **Run the pipeline directly** (useful for scripting):
   ```python
   from fake_news_module.core.pipeline import fake_news_pipeline
   result = fake_news_pipeline(r"C:\path\to\sample.pdf")
   print(result)
   ```
